# Authentication

Provide your user community with easy access to Article Galaxy using flexible authentication options.

# Okta

OKTA is the foundation for secure connections between people and technology. OKTA’s IT products use identity information to grant people access to applications on any device at any time, while still enforcing strong security protections. OKTA’s platform securely connects companies to their customers and partners.

Reprints Desk partner’s with OKTA and your organization to allow Single Sign-On into Reprints Desk’s products. Reprints Desk has created an APP within the OKTA Apps Library: https://www.okta.com/resources/find-your-apps/?keywords=reprints (opens new window).

This integration enables users to seamlessly access Reprints Desk’s software and tools without having to remember a username and password. Integration with OKTA and Reprints Desk may require some technical configuration. Additionally, there may be additional costs to configure OKTA with Reprints Desk. Please contact customersupport@reprintsdesk.com for more information.

# OpenAthens

OpenAthens is the gateway between the online world of subscription-based content and those who want to access it via their organization. Single sign-on access for end-users saves information professionals the hassle of managing multiple credentials. Using a dashboard designed by librarians, you can save time and money and audit resource usage.

Reprints Desk partner’s with OpenAthens and your organization to allow authentication into Reprints Desk services via Single Sign-On and IP Proxy. Reprints Desk can be found in the Resource catalogue within your OpenAthens administration view. This integration enables users to seamlessly access Reprints Desk’s software and tools without having to remember a username and password.

Integration with OpenAthens and Reprints Desk will require some technical configuration. Additionally, there may be additional costs to configure OpenAthens with Reprints Desk. Please contact customersupport@reprintsdesk for more information.

For further information, please read these tutorials published by OpenAthens:

# SAML 2.0

# What is it?

Reprints Desk provides a Single Sign-On capability using the SAML SSO 2.0 standard. This new method of access supplements current offerings such as web services, email and username/password options.

# Why use it?

It simplifies user management and provides end users with even more convenient access to our feature-rich online Article Galaxy system by authenticating authorized users so they can access Article Galaxy without having to remember yet another username and password.

# How does it work?

SAML SSO is an XML-based protocol that uses security tokens to securely pass information about your end user to a supplier like Reprints Desk. Information is passed between the service provider (e.g. Reprints Desk) and the identity provider (e.g. the client) to assure secure login for client users as described in this diagram.

# What is required to set this up?

Technical integration between the Reprints Desk system and a client’s system is required. Generally, this requires a member of a client’s IT team for the programming of the integration.

Specific technical requirements needed from the identity provider (E.g. the client) are:

  • Client IP Address(es) [Optional]
  • IdP’s x509 certificate public key (.cer) file
  • Url where Reprints Desk will do a HTTP-Redirect (“GET”) with the SAML Authentication Request.
  • IdP’s IssuerID:

# What are the technical details?

Reprints Desk supports SAML 2.0 Web Browser SSO Profile - Service Provider (SP) Redirect.

Supported Bindings:

  • SpToIdP: HTTP-Redirect (unsigned)
  • IdPToSp: HTTP-Post

Service Provider (SP / Reprints Desk) will provide client with:

  • Url to “POST” SAML response back to (AssertionConsumerServiceURL).
    • Staging: https://www.stg.reprintsdesk.com/saml/processhttppostassertionfromidp.aspx
    • PRODUCTION: https://www.reprintsdesk.com/saml/processhttppostassertionfromidp.aspx
  • SP’s IssuerID:
    • Staging: rpdeskspstg
    • PRODUCTION: rpdesksp

Identity Provider (IdP) to provide Reprints Desk:

  • IdP’s x509 certificate public key (.cer) file
  • Url where Reprints Desk will do a HTTP-Redirect (“GET”) with the SAML Authentication Request.
  • IdP’s IssuerID:

The Assertion.Subject.NameID must be the username (Email Address) of the user trying to SSO.

PLEASE NOTE THE REQUIRED ATTRIBUTES FOR EVERY ASSERTION (required indicates a NON-empty string value MUST be provided). ATTRIBUTES ARE CASE-SENSITIVE. Attributes MUST all be in LOWER case.

Attribute Required Value format Data type Description
firstname Yes String 50 First Name
lastname Yes String 50 Last Name
companyname String 50
